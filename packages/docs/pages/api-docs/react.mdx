# `@wizard/react` API

Source of truth: `packages/react/src/index.ts`

## Provider

### `WizardProvider`

```tsx
<WizardProvider wizard={wizard}>{children}</WizardProvider>
```

Provider usage is optional. Hooks also accept wizard instances directly.

## Hooks

- `useWizard(wizard?)`
- `useCurrentStep(wizard?)`
- `useWizardStep(wizard?, stepName)`
- `useWizardProgress(wizard?)`
- `useWizardActions(wizard?)`
- `useWizardHelpers(wizard?)`
- `useStepError(wizard?, stepName?)`
- `useWizardSelector(wizard?, selector)`

## Factory

### `createReactWizardFactory<C, E>()`

Returns:

- `defineSteps(defs)`
- `step(definition)` (supports `component` and `uiMeta`)
- `createWizard(steps, options?)`

## Router Integration

- `useSyncWizardWithRouter(wizard, options)`
- `useTanStackWizardRouter(wizard, options)`

`createWizardRouteComponent` is removed.

## Key React Types

- `ReactWizardStep`
- `StepComponent`
- `StepComponentProps`
- `ReactStepDefinition`
- `StepMetaUI`
